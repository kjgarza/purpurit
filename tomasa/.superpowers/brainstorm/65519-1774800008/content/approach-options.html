How should we structure the full overhaul?

All 3 approaches deliver the same visual result — same emerald palette, dusty rose accents, canvas textures, museum frame lightbox. The difference is how the CSS is organized.

A

Token-First (Recommended)

Rewrite the design tokens in tooling/theme.css and globals.css to the new emerald/rose palette. Then update individual components to use canvas textures, gradients, and ornamental elements. The token layer does most of the work — components inherit the new colors automatically.

Pros

  • Smallest diff — tokens cascade everywhere
  • Easy to tweak colors later (one place)
  • shadcn/ui components update automatically

Cons

  • Components with hardcoded colors need manual fixes
  • Texture/gradient layers are component-level additions
B

Component-Centric

Keep current tokens mostly intact. Override colors, textures, and gradients directly in each component file (slides, lightbox, entrance, timeline). Each component owns its visual treatment completely.

Pros

  • Full control per component
  • Can customize each slide type differently
  • No risk of unexpected cascade

Cons

  • Larger diff — duplicate color values everywhere
  • Harder to maintain consistency
  • Color changes require editing many files
C

Hybrid Layers

Rewrite design tokens for the base palette (approach A), then add shared CSS utility classes in globals.css for canvas textures, vignettes, and ornamental elements. Components compose these utilities via Tailwind classes.

Pros

  • Tokens handle color, utilities handle texture
  • Reusable texture classes across slide types
  • Clean separation of concerns

Cons

  • Slightly more CSS to maintain
  • Need to decide which effects are shared vs. unique
